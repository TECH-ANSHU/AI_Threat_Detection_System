# Dataset Directory (`dataset/`)

## Purpose
This directory stores local comma-separated values (CSV) datasets used for training and evaluating the Machine Learning threat detection model. It houses both the main training set and hold-out validation sets.

## Files Inside
- **`sample_dataset.csv`**: The primary dataset (8,000 samples) generated by `train_model.py`. Contains standard network parameters and categorical attack labels representing a balanced profile of normal (Benign) network traffic alongside simulated malicious incidents.
- **`test_data.csv`**: A hold-out evaluation dataset (2,000 samples) containing matching feature columns. Used to run local validations and compute accuracy, precision, recall, F1-scores, and draw confusion matrices.

## Features Structure
Each CSV contains the following columns:
1. `timestamp`: Timestamp of the logged traffic.
2. `source_ip`: Transmitting IP address.
3. `destination_ip`: Receiving IP address.
4. `source_port`: Network source port.
5. `destination_port`: Target port of the network packet.
6. `protocol`: Layer 4 network protocol (TCP, UDP, or ICMP).
7. `packet_length`: Byte size of individual network packets.
8. `flow_duration`: Connection active time in seconds.
9. `packet_count`: Number of packets transmitted in this flow.
10. `byte_count`: Sum of bytes in this flow.
11. `attack_type`: Categorical label (Benign, DDoS, Port Scan, Brute Force, Infiltration).
12. `label`: Binary classification label (0 = Benign, 1 = Malicious).

## System Interaction
1. **Model Training**: The script `train_model.py` generates the synthetic data, outputs both `sample_dataset.csv` and `test_data.csv` to this folder, and reads `sample_dataset.csv` to feed the training pipeline.
2. **Dashboard UI**: The Streamlit application (`app.py`) reads data from `test_data.csv` to feed the simulated live-sniffing packet capture engine, ensuring the dashboard can run realistic traffic simulations out-of-the-box.
