# Logs Directory (`logs/`)

## Purpose
This directory stores text-based log files generated by the AI-Driven Threat Detection System. These logs provide standard, high-fidelity security logging formatted for consumption by external security Information and Event Management (SIEM) tools like Splunk or ELK Stack.

## Files Inside
- **`alerts.log`**: The active text log file. Every time the system detects a network packet flagged as malicious by the machine learning engine, a detailed alert line is appended here.

## Logging Format
Log entries follow the structure:
`YYYY-MM-DD HH:MM:SS [SEVERITY] THREAT DETECTED: <Attack_Type> (Confidence: <Percent>%) - Source: <Src_IP>:<Src_Port> -> Destination: <Dst_IP>:<Dst_Port> | Protocol: <Proto> | Length: <Len> bytes`

- **[ERROR]**: Used for `CRITICAL` and `HIGH` severity threats (e.g., Infiltration, DDoS, Brute Force).
- **[WARNING]**: Used for `MEDIUM` severity threats (e.g., Port Scan).
- **[INFO]**: Used for `LOW` severity threats.

## System Interaction
1. **Writing Logs**: When the threat detection process inside `detect.py` flags traffic as malicious, it passes the findings to `alert_system.py`, which formats and writes the incident into `logs/alerts.log`.
2. **Dashboard UI**: The Streamlit interface (`app.py`) parses the last few lines of `logs/alerts.log` and renders it in a terminal emulation box on the SOC dashboard page for analyst visibility.

## Future Upgrades
- Implement automatic log rotation (e.g., rotating logs every 10MB or daily) using standard Python `RotatingFileHandler` to prevent disk exhaustion.
- Support JSON-formatted logging logs to ease ingestion into modern Cloud-native security platforms.
- Set up syslog forwarding (UDP port 514) to transmit logs directly to centralized enterprise SIEM endpoints.
